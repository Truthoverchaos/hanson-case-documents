## How to use this file

This ledger consolidates the structured facts from these existing artifacts:
- `Section 1983 Federal Claims Framework - v2.docx` (Leon_PDFs)
- `Master Timeline - Hanson Litigation.docx` (Leon_PDFs)
- `Table of Misconduct by Official.docx` (Leon_PDFs)
- "Foxes Guarding the Courthouse" X article (saved at `07_PRESS_AND_NARRATIVE/Foxes_Guarding_the_Courthouse_X_article_2025-11-21.md`)
- `Hanson_Misconduct_Matrix_1150.csv` and `Hanson_Audit_Matrix_1150.csv` (`05_FACTUAL_EVIDENCE/A_...`)
- April 4, 2025 transcript excerpt + JQC comparison table (`05_FACTUAL_EVIDENCE/...`)

Each fact has a stable ID (F###). When drafting any pleading, post, or article, **cite by ID** so revisions stay synchronized. To add a fact, append at the bottom of Section C with the next ID and date-sort during a periodic re-numbering.

**Fact record format:**
```
- **F###** · `YYYY-MM-DD` · *Actor(s)* · case-id · category
  - **Fact:** [one-sentence factual statement]
  - **Source:** [where this is documented]
  - **Supports:** [which §1983 counts]
  - **Framework v2:** [paragraph reference, or "Not yet cited"]
```

## Section F — How to Maintain This Ledger

**Adding a new fact:**
1. Append at the bottom of Section C with the next available F### ID
2. Use the standard format (date, actors, case, category, fact, source, supports, framework v2 reference)
3. Update the cross-references in Sections D and E
4. If the fact is later cited in a draft/pleading, add the citation reference to the framework_v2 line so we can track usage

**Resolving a conflict:**
1. Pick the canonical answer (or update the underlying source to make canonical)
2. Edit the relevant fact record(s) to use the canonical form
3. Remove the conflict from Section B
4. Search the workspace for other artifacts that may need updating (X article, master timeline, framework v2)

**Re-numbering (rare):**
- Avoid. Stable IDs are the whole point. Only re-number if absolutely necessary and grep for existing references first.
